HTML Doc

  1. HTML Declaration

    2. <!DOCTYPE html> declares the document to the browser as an html file.

  2. Layout

    3. The basic layout of an html file as shown and templated as below;


    In it's simplest form, an html doc can be comprised of only the head and the body.

  3. Settings/Configurations

    4. In most cases the html page settings and configurations are declared and set in the
    head section. These settings can be seen declared using meta attributes.
    The page title as well as the language used can be seen as set.
    Other configuration such as linking the page to icons, or styles etc, can also be set here.

Syntax

  1. Tags

    2. Most of the html statements starts with a tag "< >" and in some cases ends with
    a closing "</ >"tag. Some however do not have this closing tag.
    A paired tagged statement can become a parent statement, wherein nested tagged statements
    can be inserted.The nested statement is called child statement. Whereas unpaired tag statements
    cannot have child statements.

  2. Indentation

    3. It is common practice to follow conventional indentation where a parent statement can have a
    block of chicld statements each aligned accordingly. However the browser does not mind at all
    if this indentation is not observed and even successive spaces are mostly ignored.

  3. Case Sensitivity

    4. HTML is not a case sensitive programming language. Although some keywords are recommended
    to be typed uppercase and some other form of programming stanbdards can be followed.

  4. Comments

    5. Comments can be enclosed with <!--    -->. All text inside these markers are comments
    They can be read in the source code but not interpreted into the browser.

Semantics/Hierarchy

  1. Semantic Areas

    2. An html page can be semantically divided into areas. Inside the body there could be
    a header which is the topmost of the page. And there could be a navigation area or panel
    where some links or hyperlinks can be placed.

    Next to the header is the main secetion. In this main section there could be several
    sub-sections. In in these sub sections there could be some divisions.


    Outside of the main area there can be a side area.
    Some or most of the described semantic areas may be optionally used or not. But what is
    essential here is to observe the semantical hierarchy. Meaning a section is higher than a
    division and so forth. This is important specially on significances like in CSS.

  2. Headings
    3. Like in semantical areas, there is also hierarchical order when using headings.
    Again this demonstrates grouping and significances which is important for classifications
    and when creating trace backs or links.

    Headings should not be used to select default font size, but rather grouping and marking
    the hierarchical order.

Texts

Text strings can be placed in a paragraph with the common <p> tag. Continuous and
multi lines can be set.

  1. Line break

    2. Line break can be forced using the unpaired break <br> tag.Several breaks can be applied
    to form the paragraph. Empty lines or columnar space can also be made with repeated breaks.

  2. Tabs and Spaces

    3. Tabs can be set with the use of styling in CSS. To insert more white space, use the entitiy
    for space.Use the entity "nbsp" for space. Repeated use is possible, otherwise use CSS

  3. Entities

    4. Entities are special characters that cannot be displayed with regular keyboard input due to
    interaction with some syntax rules and inherent limitations. Some special characters can be
    displayed by using the entity symbol of ampersand & followed by the entity name then semicolon ;

    One example is to display the less than and greater than to display <html>, typing as it is
    would be interpreted as a valid tag.This can be done by typing & folowed by lt which is entity
    name for the lest than sign followed by semicolon ; Then type html and again by another entity
    gt for greater than sign. Search over internet for the list of these entities.

  4. Lorem

    5. Lorem is a way to add dummy words to easily populate a test paragraph.
    To have 20 random latin words in html, type Lorem followed by the number of words.
    Typing Lorem20 then press tab, will generate 20 latin words.

  5. Immediate Styling

    6. Immediate styling is possible but not recommended to modify the appearance of some words
    in a line or sentence. These styling tools are called emphasis or em's. The word or words needs
    to be enclosed with a pair of em tags. See Example
